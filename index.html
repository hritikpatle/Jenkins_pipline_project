CI/CD Pipeline with Jenkins, Selenium & SonarCloud

Assignment 11 - DevOps Automation

1. Jenkins Pipeline Setup

A Jenkinsfile is created and pushed to the GitHub repository. This file defines the stages of the pipeline including build, test, SonarCloud scan, and packaging.

2. Selenium Testing

Selenium test scripts are included in the project. Jenkins runs these tests automatically to validate the web application functionality.

3. SonarCloud Scanning

SonarCloud is integrated into the pipeline to analyze code quality. The pipeline uses a secure token to authenticate and send results to SonarCloud.

4. Package Building

The pipeline builds the application using Maven or another build tool. The output is a deployable package such as a .jar or .war file.

5. Results

  • Package: Available in Jenkins artifacts
  • SonarCloud Report: Viewable on the SonarCloud dashboard
  • Test Results: Displayed in Jenkins test reports
